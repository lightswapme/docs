The LightSwap Partner API lets approved partners create crypto swaps from their own backend. Your application keeps the user experience, and LightSwap returns quotes, deposit instructions, and status updates through a small REST API.

<Note>
  Partner API keys are server-side credentials. Do not expose them in browser code, mobile apps, widgets, public repositories, or client-side logs.
</Note>

## Base URL

```text
https://lightswap.me/api/partner/v1
```

## Integration model

The MVP flow is exact-input and floating-rate:

<Steps>
  <Step title="List supported assets">
    Use `GET /asset` to load supported assets, networks, decimals, logos, and memo requirements.
  </Step>
  <Step title="Validate the payout address">
    Use `POST /address` before creating a swap so your user can fix address mistakes early.
  </Step>
  <Step title="Create a quote">
    Use `POST /quote` with the send asset, receive asset, and send amount. Quotes are short-lived.
  </Step>
  <Step title="Create the swap">
    Use `POST /swap` with the quote ID, destination address, and an `Idempotency-Key` header.
  </Step>
  <Step title="Poll status">
    Use `GET /swap/{id}` until the swap reaches a terminal status.
  </Step>
</Steps>

## Public API scope

The Partner API MVP includes:

- Active asset discovery.
- Destination address validation.
- Exact-input quote creation.
- Swap creation from a quote.
- Swap status lookup.
- Partner attribution through `externalId` and constrained `metadata`.

It does not include webhooks, browser keys, public dashboards, refund endpoints, or per-request custom fees.
