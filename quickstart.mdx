---
title: "Quickstart"
description: "Create a quote, create a swap, and poll its status with the LightSwap Partner API."
---

This quickstart shows the minimum backend flow for creating a LightSwap swap.

## Prerequisites

- An active partner API key.
- A backend service that can keep the key private.
- A destination wallet address for the asset your user wants to receive.

Set your key as an environment variable on your backend:

```bash
export LIGHTSWAP_PARTNER_KEY="lsk_live_..."
```

## 1. List assets

Use the asset list to build your pair selector and to confirm network names.

```bash
curl "https://lightswap.me/api/partner/v1/asset?symbol=USDT" \
  -H "x-api-key: $LIGHTSWAP_PARTNER_KEY"
```

The response includes `network`, `decimals`, `contractAddress`, `logoUrl`, `canSend`, `canReceive`, and `requiresMemo`.

## 2. Validate the destination address

Validate the payout address before creating the swap.

```bash
curl -X POST "https://lightswap.me/api/partner/v1/address" \
  -H "content-type: application/json" \
  -H "x-api-key: $LIGHTSWAP_PARTNER_KEY" \
  -d '{
    "network": "Tron",
    "token": "USDT",
    "address": "TNDzfERDpxLDS2w1q6yaFC7pzqaSQ3Bg3r",
    "memo": null
  }'
```

For the MVP, destination memo is not supported. Send `null` or omit it.

## 3. Create a quote

Create a short-lived floating quote. Send `from.amount` and omit `to.amount`.

```bash
curl -X POST "https://lightswap.me/api/partner/v1/quote" \
  -H "content-type: application/json" \
  -H "x-api-key: $LIGHTSWAP_PARTNER_KEY" \
  -d '{
    "from": {
      "network": "Bitcoin",
      "token": "BTC",
      "amount": "0.01"
    },
    "to": {
      "network": "Tron",
      "token": "USDT"
    },
    "externalId": "partner-order-123",
    "metadata": {
      "customerId": "u_123"
    }
  }'
```

The response returns a `pq_` quote ID and an `expiresAt` timestamp. Create the swap before that time.

## 4. Create a swap

Use the quote ID and an idempotency key. Reusing the same idempotency key with the same body returns the same swap.

```bash
curl -X POST "https://lightswap.me/api/partner/v1/swap" \
  -H "content-type: application/json" \
  -H "x-api-key: $LIGHTSWAP_PARTNER_KEY" \
  -H "Idempotency-Key: partner-order-123-create" \
  -d '{
    "quoteId": "pq_aaaaaaaaaaaaaaaaaaaaaaaa",
    "destination": {
      "address": "TNDzfERDpxLDS2w1q6yaFC7pzqaSQ3Bg3r",
      "memo": null
    },
    "notificationEmail": "optional@example.com"
  }'
```

The response returns:

- `id`: the LightSwap swap ID.
- `status`: the current public status.
- `deposit`: the address, amount, asset, network, memo, and deadline for the user deposit.
- `payout`: the destination details.
- `quote`: the partner quote ID and route timing estimate.

## 5. Poll status

Poll every 30 seconds while the status is not terminal.

```bash
curl "https://lightswap.me/api/partner/v1/swap/AB12CD34EF56" \
  -H "x-api-key: $LIGHTSWAP_PARTNER_KEY"
```

Terminal statuses are `completed`, `refunded`, `failed`, and `expired`.

<Tip>
  Store the LightSwap swap ID with your own order ID. It is the value support needs when checking a specific swap.
</Tip>
